## Section 12 ÔÇö Gap Analysis and Fix Suggestions

### GAP 1 ÔÇö Total Profit Understated by ~$99.7M (CRITICAL)
**Items affected:** #18, #19, #20, #21, #22, #25, #55, #58, #59 ÔÇö all KPIs, profit, IRR

**Root cause:** The app's profit formula is:
```
Profit = Revenue Ôłĺ GST_on_Revenue Ôłĺ Total_Cost
       where Total_Cost INCLUDES GST_on_Costs as a line item
```
This means the app deducts BOTH GST on costs ($72M) AND GST on revenue ($90M). But GST on costs is a cash outflow that is 100% recovered from the ATO as Input Tax Credits (ITC). The developer's real GST burden is only the **net GST** (GST on revenue minus ITC Ôëł $18M).

**Fix (Option A ÔÇö recommended, matches Excel):**
Remove GST on Costs from the `totalCost` formula in `engine/index.ts`. Replace with Net GST only:
```typescript
// In engine/index.ts, change:
const totalCost = totalLand + totalStampDuty + totalBuildCosts + totalContingency +
  totalSeniorFinCosts + totalLandLoanFinCosts + totalMezzFinCosts + totalOtherFin +
  standardCosts + totalGSTOnCosts + totalMarketing + commissions.total + totalPMFees;

// To:
const netGST = totalGSTOnRevenue - totalGSTOnCosts; // positive = payable to ATO
const totalCost = totalLand + totalStampDuty + totalBuildCosts + totalContingency +
  totalSeniorFinCosts + totalLandLoanFinCosts + totalMezzFinCosts + totalOtherFin +
  standardCosts + netGST + totalMarketing + commissions.total + totalPMFees;

// And update profit formula:
const totalProfit = totalSettlementsRevenue + totalRentalIncome + totalOtherIncome - totalCost;
// (no separate GST deduction needed ÔÇö net GST is already in totalCost)
```
Also update the feasibility object returned: replace `gst: totalGSTOnCosts` with `gst: netGST` for dashboard display (matching Excel's single GST line = net payable).

### GAP 2 ÔÇö Senior Line Fee on Facility Limit vs Drawn Balance (HIGH)
**Items affected:** #14, #31, #32, #48, and all KPIs cascaded through higher finance costs

**Root cause:** In `engine/funding.ts` line ~450:
```typescript
periodFees += periodInterest(seniorLimit, senior.lineFeePercent, days, daysPerYear);
```
`seniorLimit` is the **facility limit ($767M)**, not the drawn balance. Over 41 months this generates ~$56M in line fees. The Excel charges line fee on the **undrawn committed balance** (facility limit minus current drawn balance), which averages ~$383M for approximately 20 active draw-down months, yielding ~$25.7M.

**Fix:**
```typescript
// Change line fee base from facility limit to undrawn balance:
const undrawnBalance = Math.max(0, seniorLimit - snrOpenBalance);
periodFees += periodInterest(undrawnBalance, senior.lineFeePercent, days, daysPerYear);
```
This will reduce senior fees from $57.4M to ~$25-28M, aligning with the Excel's $29.5M (residual difference may come from exact balance timing).
